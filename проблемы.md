# Известные проблемы проекта

## Маркер съезжает при zoom

**Описание:** При изменении уровня zoom карты кастомный маркер (пин) смещается относительно своей географической точки. При максимальном zoom (крупном) маркер совпадает с точкой, при отдалении — уходит.

**Причина:** Yandex Maps API 3.0 использует CSS transforms для позиционирования маркеров на карте. Кастомный `transform: translate()` в компоненте маркера конфликтует с внутренними трансформациями API.

**Решение:** 
1. Заменить `transform: translate(-25px, -57px)` на `margin-left: -25px; margin-top: -57px` в `.marker-pin`.
2. Перенести hover/click анимацию (`transition` + `transform: translateY`) на внутренний элемент `.marker-pin__shape`, чтобы анимация не влияла на глобальное позиционирование контейнера.

**Почему это работает:**
- Yandex Maps API 3.0 использует CSS transforms для позиционирования маркеров на карте (для производительности через GPU).
- `margin` не создает отдельного compositing layer и не конфликтует с внутренними трансформациями API.
- Анимация hover/click теперь применяется к внутреннему элементу `.marker-pin__shape`, а не ко всему контейнеру маркера.

**Итоговый CSS:**
```css
.marker-pin {
  margin-left: -25px;
  margin-top: -57px;
}

.marker-pin__shape {
  transition: transform 160ms ease;
}

.marker-pin--hover .marker-pin__shape {
  transform: translateY(-1px);
}

.marker-pin--click .marker-pin__shape {
  transform: translateY(1px);
}
```

**Компоненты:**
- `src/components/Marker.vue` — стили маркера
- `src/components/MapView.vue` — позиционирование маркеров
