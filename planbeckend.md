# planbeckend

## Цель
Сделать backend на 1C-Bitrix для текущего Vue-проекта:
- хранение контента в одном инфоблоке
- админка для редактирования данных
- API для чтения данных фронтом
- токен-защита API

## Контекст проекта
Текущий проект:
- Vue 3 + Vite + Pinia + Router + Tailwind
- сейчас данные лежат в `src/data/mock.ts`
- на фронте есть категории `projects`, `travel`, `sport`
- у `projects` есть задачи
- карта и карточки уже готовы, нужно только заменить источник данных

## Принятое решение
Использовать 1C-Bitrix как backend и админку.
Фронт оставляем на Vue и подключаем к Bitrix через API.

Важно: использовать **один инфоблок**.
Табы на фронте соответствуют разделам инфоблока: `projects`, `travel`, `sport`.

Иерархия разделов:
- `projects`, `travel`, `sport` - корневые разделы инфоблока
- внутри `travel` и `sport` - элементы
- внутри `projects` сначала идут вложенные разделы с названием конкретного проекта
- уже внутри каждого проектного раздела лежат элементы проекта

## Архитектура
- Новый сайт на Bitrix
- 1 инфоблок с корневыми разделами под категории
- проектные записи организованы через вложенные разделы внутри `projects`
- API отдает нормализованный JSON
- фронт получает данные по токену

## Структура данных

### Один инфоблок
Поля:
- `NAME` -> title
- `PREVIEW_TEXT` -> description
- `DETAIL_PICTURE` или множественное поле для images
- свойства:
  - `COORDINATES_LAT`
  - `COORDINATES_LNG`
  - `CITY`
  - `DATE`
  - `STATUS`
  - `TAGS`
  - `IMAGES`
  - `SORT`
  - `CATEGORY` если нужен явный дубль для API, но источник истины - раздел

Разделы:
- `projects`
  - вложенные разделы с названием проекта
    - элементы проекта
- `travel`
  - элементы
- `sport`
  - элементы

Если у проекта есть задачи, лучше хранить их как элементы внутри проектного раздела, а сам проект делать разделом-контейнером.

Отдельный инфоблок `tasks` не нужен в этой схеме.
Если понадобится отдельная логика для задач, её лучше строить через тип элемента/свойство внутри проектных разделов и выборку по API.

## API
Сделать read-only JSON API для фронта.

Рекомендуемые эндпоинты:
- `GET /api/v1/categories`
- `GET /api/v1/sections?category=projects`
- `GET /api/v1/items?sectionId=...`
- `GET /api/v1/projects`
- `GET /api/v1/travel`
- `GET /api/v1/sport`
- `GET /api/v1/filters`

Формат ответа должен совпадать с текущими типами фронта:
- `id`
- `title`
- `description`
- `coordinates`
- `category`
- `date`
- `city`
- `tags`
- `status`
- `images`
- `tasks`

Для `projects` API должно возвращать:
- список проектных разделов
- внутри каждого раздела - элементы проекта

Для `travel` и `sport` API может возвращать:
- раздел категории
- список элементов внутри него

## Авторизация API
Нужен токен для чтения данных.
Минимальный вариант:
- токен в `Authorization: Bearer ...`
- проверка токена на стороне Bitrix
- публичная выдача данных запрещена

## Админка
Нужно обеспечить удобное редактирование:
- список элементов инфоблока
- создание/редактирование карточек
- загрузка изображений
- редактирование координат, дат, статусов, тегов
- создание корневых и вложенных разделов
- перенос элементов между разделами

## Что нужно сделать в Bitrix
1. Создать новый сайт Bitrix.
2. Создать тип инфоблоков и сами инфоблоки.
3. Настроить свойства.
4. Сделать API-слой через D7/контроллеры.
5. Добавить токен-проверку.
6. Проверить выдачу JSON.
7. Подготовить фронт к замене `mock.ts` на запросы API.

## Что менять во Vue-проекте
- заменить `src/data/mock.ts` на API-клиент
- обновить store, чтобы данные грузились из Bitrix
- оставить текущую UI-логику без переписывания

## Рекомендуемый порядок работ
1. Зафиксировать структуру инфоблоков.
2. Создать инфоблоки в Bitrix.
3. Реализовать API.
4. Подключить токен.
5. Подключить фронт.
6. Протестировать карту, фильтры, карточки и задачи.

## Критерии готовности
- данные редактируются в админке Bitrix
- фронт получает данные через API
- токен проверяется
- проекты, travel, sport и tasks отображаются корректно
- фильтры и карта работают на реальных данных

## Примечания
- Не переносить логику интерфейса в Bitrix без необходимости
- Не делать лишнюю абстракцию, нужен минимальный рабочий backend
- Если будут нюансы по версиям Bitrix или существующей установке, уточнить до реализации
